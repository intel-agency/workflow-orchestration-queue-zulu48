# AI Repository Summary

## Project Overview

**Name:** workflow-orchestration-queue (OS-APOW)
**Type:** Headless Agentic Orchestration Platform
**Repository:** intel-agency/workflow-orchestration-queue-zulu48

## Purpose

Transforms GitHub Issues into autonomous execution orders, enabling AI-driven development without human-in-the-loop intervention.

## Technology Stack

| Component | Technology |
|-----------|------------|
| Language | Python 3.12+ |
| Web Framework | FastAPI + Uvicorn |
| Validation | Pydantic |
| HTTP Client | HTTPX (async) |
| Package Manager | uv |
| Containerization | Docker, Docker Compose, DevContainers |
| Agent Runtime | opencode CLI with GLM-5 |

## Architecture

### 4-Pillar Architecture

1. **The Ear (Notifier)** - `src/notifier_service.py`
   - FastAPI webhook receiver
   - HMAC SHA256 signature validation
   - Event triaging and WorkItem generation

2. **The State (Queue)** - `src/queue/github_queue.py`
   - GitHub Issues as distributed state
   - Label-based status machine
   - Assign-then-verify concurrency control

3. **The Brain (Sentinel)** - `src/orchestrator_sentinel.py`
   - Background polling with jittered backoff
   - Shell-bridge execution via `devcontainer-opencode.sh`
   - Heartbeat comments for long-running tasks

4. **The Hands (Worker)** - DevContainer
   - Isolated Docker environment
   - opencode CLI agent runtime

## Build Commands

```bash
# Install dependencies
uv sync --dev

# Run tests
uv run pytest tests -v

# Lint code
uv run ruff check src tests

# Format code
uv run ruff format src tests

# Type check
uv run mypy src

# Build Docker image
docker build -t workflow-orchestration-queue .
```

## Run Commands

```bash
# Start notifier service
uv run uvicorn src.notifier_service:app --reload

# Start sentinel
uv run python -m src.orchestrator_sentinel

# Docker Compose
docker-compose up -d
```

## Security

- HMAC SHA256 webhook signature validation
- Ephemeral credentials (never written to disk)
- Credential scrubbing for public logs
- Network isolation for worker containers
- Resource constraints (2 CPUs, 4GB RAM)
